## 批量翻译功能 💰

我们的第一个 Beta 功能解决了翻译用户最大的痛点：**API 成本和速率限制**。

### 我们要解决的问题

想象一下阅读一篇 20 段的文章。传统翻译方式：

- 📤 **发送 20 次单独的 API 请求**到您的 LLM 提供商
- 💸 **5 倍的成本** - 额度快速消耗
- 🚫 **速率限制陷阱** - 不断遇到"请求过多"错误
- 😤 **令人沮丧的中断** - 阅读流程被打断

是不是很熟悉？

### 解决方案：智能批处理

<Callout type="tip">
**批量翻译**将多个段落组合成单个 API 请求。质量相同，成本大幅降低。
</Callout>

这就是魔法所在：

| 批处理前 | 批处理后 | 节省 |
|---------|---------|-----|
| 20 段 = 20 次 API 调用 | 20 段 = 3-4 次 API 调用 | **减少 60-80% 调用** |
| 每月 ¥70 API 成本 | 每月 ¥14-28 API 成本 | **每月节省 ¥42-56** |
| 频繁触发速率限制 | 轻松保持在限制内 | **不再中断** |

![Batch Translation](/blog/batch-translation-feature/batch-translation.png)

### 真实影响

**成本节省示例：**

如果您每月翻译 100 篇文章（平均每篇 20 段）：
- **不使用批处理**：2,000 次 API 调用
- **使用批处理**：300-400 次 API 调用
- **节省**：减少 60-80% = **API 成本从 ¥350 降至 ¥70-140/月**

<Callout type="warn">
**免费层用户必看**：许多提供商限制每分钟 3-5 次请求。批量翻译让您在这些限制内翻译 5-10 倍的内容！
</Callout>

## 如何启用 🎮

开始使用不到 60 秒：

1. **打开陪读蛙设置**（点击扩展图标 → 设置）
2. **导航到功能** → **Beta 体验**
3. **开启"启用 Beta 体验"**
4. **找到下方的"批量翻译"**部分
5. **自定义批处理设置**（或使用智能默认值）：

| 设置 | 默认值 | 作用 |
|-----|-------|------|
| **最大字符数** | 1000 | 批量处理的字符数 |
| **最大段落数** | 4 | 每批处理的段落数 |

**就是这样！** 批量翻译现在会在您翻译网页时自动工作。

<Callout type="tip">
**专业建议**：从默认设置开始（1000 字符，4 段）。它们适用于大多数内容。根据需要稍后调整。
</Callout>

## 微调您的设置 ⚙️

根据内容类型匹配批处理设置：

| 内容类型 | 推荐设置 | 原因 |
|---------|---------|------|
| **新闻文章** | 1500 字符，6 段 | 语言简单，最大化批处理 |
| **技术文档** | 800 字符，3 段 | 术语复杂，需要精确性 |
| **社交媒体** | 500 字符，2 段 | 短帖子，快速处理 |
| **研究论文** | 1000 字符，4 段 | 平衡质量与效率 |

**自由实验！** 您可以随时根据结果进行调整。
