## 首发功能:批量翻译 💰

我们的第一个 Beta 功能是 **批量翻译** - 一项能够帮您节省 60-80% API 成本并避免令人沮丧的速率限制错误的颠覆性功能。

### 工作原理

陪读蛙不再逐个翻译段落,而是智能地将多个段落组合在一起,通过单个请求发送给您的 AI 提供商进行翻译。

**之前**: 20 个段落 = 20 次单独的 API 请求
**之后**: 20 个段落 = 3-4 次批量请求

API 调用减少了 **60-80%**!

### 您将感受到的实际好处

- **节省 60-80% 的 API 成本** - 如果您每月在翻译上花费 ¥70,这可以降至 ¥14-28。您的 API 额度使用时长延长 3-5 倍
- **绕过速率限制** - 许多免费层 API 限制为每分钟 3-5 次请求。批量翻译让您在这些限制内翻译更多内容,不会遇到"超出速率限制"错误
- **不间断阅读** - 不再因用尽配额而在文章中途停下
- **智能批处理** - 可配置的限制让您在成本节省和翻译粒度之间取得平衡

如果您在翻译长篇文章时曾遇到"超出速率限制"的错误,批量翻译通过大幅减少 API 调用次数来解决这个问题。

### 如何启用 Beta 体验与批量翻译

1. 打开陪读蛙设置
2. 导航到 **功能** → **Beta 体验**
3. 将 **启用 Beta 体验** 切换为开启
4. 滚动到 **批量翻译** 部分
5. 自定义您的偏好:
   - **最大字符数**:批量处理的字符数(默认:1000)
   - **最大段落数**:每批处理的段落数(默认:4)

启用后,批量翻译会在您翻译网页时自动工作!

### 使用场景

这个功能在以下情况下特别有用:

- **使用免费层 API** - 许多 LLM 提供商对免费层有严格的速率限制;批量翻译让您在这些限制内翻译更多内容
- **预算有限** - 每月节省 60-80% 的 API 费用
- **阅读长篇文章** - 翻译新闻文章、博客文章和文档时不会触发速率限制错误
- **研究工作** - 在 API 配额内翻译多篇研究论文
- **重度用户** - 如果您每天翻译大量内容,成本节省会快速累积

### 根据需求微调

可配置的限制让您在速度和质量之间取得平衡:

- **更高的限制**(例如,2000 字符,8 个段落)- 最大速度,适合简单内容
- **更低的限制**(例如,500 字符,2 个段落)- 更精细的控制,适合复杂或技术性内容

尝试找到最适合您阅读习惯的设置!
