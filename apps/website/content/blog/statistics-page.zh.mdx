## 您将看到什么 👀

### 概览指标

四个强大的指标卡片在您打开统计页面时就会映入眼帘：

| 指标 | 展示内容 | 为什么重要 |
|------|---------|-----------|
| **原始请求数量** | 没有批处理您*本该*发起的 API 调用数 | 您的基准线 |
| **批量请求数量** | 您*实际*发起的 API 调用数 | 您的真实 API 使用量 |
| **节省的请求** | 两者之间的差值（您的效率提升） | 节省的证明 |
| **趋势指示器** | ↗️ 增加或 ↘️ 减少（相对于上一周期） | 即时发现模式 |

每张卡片都显示**与上一周期的比较** - 让您一眼就能看出情况是变好还是变坏。

<Callout type="tip">
**绿色箭头是你的朋友**：在"节省的请求"上看到 ↗️ 意味着您变得更高效了！
</Callout>

## 深入了解您的数据 🔍

### 比较时间周期

在不同视图之间切换以满足您的分析需求：

| 视图 | 最适合 |
|------|-------|
| **5 天** | 发现最近的变化 |
| **7 天** | 每周模式（默认） |
| **30 天** | 月度趋势 |
| **60 天** | 长期分析 |

### 理解您的节省

显著显示的**节省百分比**告诉您：

```
节省 % = (原始请求 - 批量请求) / 原始请求
```

**翻译一下？** 如果您看到 **75%**，意味着您用 4 次 API 调用完成了原本需要 16 次的工作。

<Callout type="info">
**典型节省范围**：大多数用户看到 60-85% 的节省。如果您低于 50%，请考虑在设置中增加批处理大小。
</Callout>

## 数字的含义 💭

让我们分析一个真实例子：

---

**您的统计数据显示：**
- 原始请求数量：**500**
- 批量请求数量：**100**
- 节省的请求：**400**
- 平均节省：**80%**

**这意味着什么：**

如果您的提供商每次请求收费 ¥0.07：

| 场景 | 成本 |
|------|------|
| **不使用批处理** | 500 × ¥0.07 = **¥35.00** |
| **使用批处理** | 100 × ¥0.07 = **¥7.00** |
| **您节省了** | **¥28.00** (80%) |

---

如果一个月大量使用，这样计算？**节省快速累积。**

<Callout type="tip">
**专业技巧**：在调整批处理设置前后截图您的统计数据，以查看哪种配置能给您带来最佳结果。
</Callout>
